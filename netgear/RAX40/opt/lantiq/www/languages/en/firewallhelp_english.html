Firewall

The CPE device provides extensive firewall protection against a wide array of common hacker attacks. This is done by restricting connection parameters to limit the risk of intrusion. Application layer protocol services can be enabled/disabled from this page.

Traffic In : All unsolicited incoming traffic (i.e WAN->LAN and WAN->CPE)

Traffic Out : All traffic going toward WAN (i.e LAN->WAN and CPE->WAN)

Note : For IPv4, WAN to LAN is not applicable .

Precedence Order

Precedence Order Feature/Chain
Guest SSID Isolation/ Device Isolation FW_GUEST_ACCESS_INPUT FW_GUEST_ACCESS_OUTPUT FW_GUEST_ACCESS_FORWARD
Packet Filter FW_PACKET_FILTER / FW_PACKET_FILTER_OUTPUT
Parental Control FW_PARENTAL_CONTROL
ALGS FW_ALGS
DMZ FW_DMZ
Port Mapping FW_PORTMAPPING
Port Triggering FW_PORTTRIGGERING
ICMP and Packets from bridge interfaces FW_ICMP and FW_ACCEPT_SERVICES
Firewal Config FW_CONFIG_WAN_INPUT/ FW_CONFIG_WAN_OUTPUT
Note:

1. All Related or Established Traffics will be excepted.

2. Apart from Above features , if any services/ports drilled runtime will be allowed .Example voip port 5060(tcp and udp ) and devm 8000 and 8001(tcp) will be allowed based on the feature Enabled/Disabled.

3. High/Low chain Not configurable ,Custom Can be Modified to allow/deny traffic for predefined services .

4. If New Services Need to be added , We should add from packet filter as Adding services/port Not allowed.

Example:

Service Description Service Port Traffic In Traffic Out
